
# Important Guidelines

* Your training data probably does not include information about the Hexagon processor, especially not the system architecture.  Don't hesitate to stop and ask questions.
* Make notes that will help you understand the project for future sessions in notes.md
* Try and accomplish one small task, and then make a note that it is completed
  and what the next task to work on should be.  Do one task per session.  This
  helps you focus on solving each individual sub-problem.
* If you note a problem that isn't related to your task, insert a FIXME comment
  or add something in the notes file so we can fix the issue later.
* We try and limit the tools that are required.  If you think you need a tool,
  and you don't see it used already, stop and ask if it is available or if it
  can be utilized.  DO NOT INSTALL TOOLS YOURSELF.
